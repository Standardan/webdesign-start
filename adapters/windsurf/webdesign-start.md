Read the file `webdesign-start/SKILL.md` in this project (locate the `webdesign-start` folder if it is not at the root) and follow its workflow exactly, starting at Phase 0:

1. **Intake** — harvest what the user already said; never re-ask known answers.
2. **Discovery** — adaptive batches of 3-4 multiple-choice questions (see `references/discovery.md`).
3. **Research** — find real example websites, present as links with "why this matches you" notes, collect reactions (see `references/research.md`).
4. **Brief** — write `DESIGN-BRIEF.md`, show a digest, get explicit approval. Hard gate.
5. **Build** — implement against the brief and `references/build-standards.md` + `references/ux-rules.md`.
6. **Review** — self-audit against the brief and the pre-delivery checklist; report honestly.

Do not skip the questionnaire, the research phase, or the brief-approval gate.
