# /webdesign-start — Guided web design discovery & build

Read the file `webdesign-start/SKILL.md` in this project (if it is not at the project root, locate the `webdesign-start` folder — it may live in `.cursor/`, a `skills/` directory, or the repo root) and follow its workflow exactly, starting at Phase 0.

Summary of what you are signing up for: run a short adaptive discovery questionnaire (batches of 3-4 multiple-choice questions), research real example websites and present them as links for the user to react to, synthesize an approved `DESIGN-BRIEF.md`, and only then build the site against that brief, finishing with an honest self-review.

Do not skip the questionnaire, the research phase, or the brief-approval gate — the entire value of this command is removing ambiguity before code is written.
